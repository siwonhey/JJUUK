# [Interactive Posture Companion(Posture Buddy)] - 실시간 인터랙티브 캐릭터 반응형 자세 유도 프로그램

## 1. 목표
컴퓨터 작업 중, 사용자의 자세 변화를 웹캠으로 감지하여, 설정한 바른 자세를 벗어났을 경우 각각 거북이, 기린 영상이 등장해 자연스럽고 재밌게 자세를 교정하도록 유도하는 데스크탑 프로그램 제작


## 2. 핵심 기능(기능 요약)
1) 사용자는 시작 시 바른 자세를 캘리브레이션한다.(얼굴 위치, 얼굴 크기, 얼굴 높이)
2) 1에서 측정한 기준값과 비교하여 자세 이상 여부를 판단한다.
3) 이상 여부 조건 충족 시 애니메이션이 실행된다.
- **거북이 등장**: 미리 촬영된 얼굴보다, 사용자의 얼굴이 카메라에 가까이 오는 경우
- **기린 등장**: 미리 촬영된 얼굴 위치보다, 사용자의 얼굴 위치가 아래로 내려가는 경우 등
4) **자세 복귀 시**: 애니메이션이 다시 화면 밖으로 사라진다.
5) 다시 정해진 자세를 벗어나는 경우 이를 반복하여, 바른 자세를 유도한다.


## 3. 조건
- 맥북, 윈도우에서 모두 구동 가능해야 함.(동시 지원 가능이 BEST, 어려울 경우 따로 시스템 마련- 메뉴바(맥),시스템 트레이(윈도우))
- 프로그램 실행 방법은 맥북의 경우, 화면 상단 아이콘 바에 프로그램 아이콘을 두어 켜고 끌 수 있도록 한다. (윈도우는 방법 마련을 구상하라)
    - 켜기/끄기, 자세 기본값 재측정이 수월해야 함, 민감도 조절
- 이후 제작 완료 후 배포할 예정임을 염두에 두라.


## 4. 사용자 플로우
### [FLOW 1] — 프로그램 실행
:사용자가 프로그램 실행.

프로그램은 메뉴바(맥) 시스템 트레이(윈도우)에 상주
- macOS:상단 메뉴바 아이콘 유지
- Windows:우측 하단 시스템 트레이 아이콘 유지

기능: 켜기/끄기, 재측정, 민감도 조절

### [FLOW 2] — 자세 측정 (Calibration)
: 웹캠에서 **사람의 얼굴·상체 랜드마크를 추적해서 기준 자세 대비 변화량**을 보는 방식

- 사용자는 바른 자세 상태에서 “측정 시작” 버튼 클릭.

### [FLOW 3] - 기준 자세(Baseline) 저장. 이후 현재 상태와 비교한다.

즉 핵심은 절대값 판정이 아닌 “초기 상태 대비 얼마나 변했는가”
처음 시작할 때: “정상 자세”를 기준으로 저장한다.

### [FLOW 4] - 자세 변화량 계산
여기서 거북목, 굽은 허리 판별



## 5. 자세 감지 로직
1) 거북목 감지
조건: 사용자의 얼굴 크기가 baseline보다 커질 경우.( 얼굴이 카메라에 가까워진 상태)

(계산 방식 예) currentFaceWidth > baselineFaceWidth * threshold
예시: 1.15배 이상 접근 시 감지.

2) 굽은 허리 감지
조건: 사용자 얼굴 중심 Y 좌표가 아래로 내려갈 경우.(앉은 자세가 무너진 상태)

(계산 방식 예) currentFaceY > baselineFaceY + threshold

## 6. 캐릭터 시스템
: 상태 변화시에만 애니메이션

### A. 거북목 방지 거북이
트리거: 거북목 감지 시.

[연출]
거북이가 화면 밖에서 헤엄치며 등장하여 중앙까지 도달.
자세가 돌아올 때까지 대기(자세 복귀 시 거북이가 다시 화면 밖으로 퇴장)


### B. 굽은 자세 감지 방지 기린
트리거: 굽은 자세 감지 감지 시.

[연출]
기린이 화면 밖아래에서 등장하여 중앙까지 목이 길어짐.
자세가 돌아올 때까지 위치 목이 길어진 상태 대기(자세 복귀 시 기린이 다시 화면 밖으로 퇴장)


** 중요 **
이 프로젝트는 의료 수준의 자세 교정 프로그램이 아니라,
사용자가 자연스럽게 자세를 인지하도록 만드는
인터랙티브 데스크 컴패니언 경험에 가깝다.

따라서:
완벽한 자세 판별보다 안정적인 동작,낮은 리소스 사용량, 자연스러운 캐릭터 반응을 우선한다.


### Character Implementation Strategy — Turtle & Giraffe
Interactive Posture Companion(Posture Buddy) 프로젝트의
거북이/기린 캐릭터 구현 전략을 정의한다.

핵심 목표:
* 낮은 CPU/GPU 사용량
* 자연스러운 인터랙션
* 항상 켜져 있는 데스크탑 오버레이 환경 대응

# 전체 구현 전략

## 1. Turtle
“살아있는 생물 느낌” 중심

### 추천:
* 투명 WebM 루프 애니메이션
* 또는 경량 GLB

### 목표
사용자가 앞으로 목을 빼는 순간,
거북이가 화면 밖에서 등장하여
살아있는 생물처럼 천천히 헤엄쳐 나온다.

자세 복귀 시:다시 화면 밖으로 퇴장한다.

### 상태
- entering: 화면 밖 → 중앙 근처 이동
- idle: 제자리에서 부유감 있는 loop motion
- exiting: 화면 밖으로 퇴장

---
## 2. Giraffe
“목이 길어지는 연출” 중심

추천:
* 기린 머리 PNG/WebM
* CSS 기반 목 stretch
* transform animation

---

### 구현 방식(다른 방식 추천 시, 제시하라)
- Position Animation: CSS transform translateX()
- Floating Motion: CSS keyframes 또는 GSAP
- Rendering: HTML video (alpha WebM)

---
### 구조

```text
[기린 머리]
     │
[늘어나는 목]
     │
(화면 아래)
```

몸통은 표시하지 않는다.


### Motion 구조

- **entering**: 아래에서 상승
- **stretching**: 목 길이 증가
- **idle**: 머리 미세 움직임
- **exiting**: 아래로 하강



### 캐릭터는 사용자의 작업 화면 위에 자연스럽게 표시되는
투명 오버레이 형태로 동작해야 한다.

### 성능 전략
본 프로그램은 장시간 백그라운드 실행을 전제로 하며, 사용자의 기존 작업 환경을 방해하지 않는 것을 최우선으로 한다.
